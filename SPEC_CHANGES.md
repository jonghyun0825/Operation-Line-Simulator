# 명세서 변경 이력 (v3 → v3.1)

`가상조립라인_프로젝트명세서.md.pdf`(원본, v3)는 PDF라 이 저장소의 도구로 직접 편집할 수 없다.
이 문서는 v3 대비 적용된 7건의 수정 요구사항을 조항별로 정리한 변경 이력이며, **원본 PDF와 함께
읽어야 하는 보충 문서**다. 원본에서 여기 언급된 조항은 이 문서의 내용이 우선한다.

## F1. 생산 조건 설정 및 시작

- **레일 속도 조절 UI 삭제.** 조건 패널은 박스 수·온도·헤드 속도 3항목만 남는다.
- `POST /api/lot` body에서 `conveyor_speed` 제거: `{boxes, temperature, head_speed}`.

## F2. 2D 조립라인 애니메이션 (SVG)

- **기계 묘사 예외 조항 추가**: "두 기계는 완전히 동일한 형태" 규칙에 예외를 둔다. 베이스·칼럼의
  형태·크기·위치는 M1/M2가 동일하되, **체결 헤드의 가로 장착 위치만 슬롯 중심 기준 좌우 대칭으로
  오프셋**될 수 있다(ST1은 왼쪽, ST2는 오른쪽). 헤드와 칼럼을 시각적으로 잇는 마운트 암을 추가해도 된다.
  오프셋 값은 나사 지점 좌표와 반드시 같은 상수를 참조해야 하며(아래 참조), 상하 운동 동기화 규칙은
  변경 없이 유지된다.
- **나사 지점 = 헤드 정렬 기준.** 나사 표시 위치(반제품 위 막대 폭의 20%/80% 지점)가 source of truth이며,
  헤드(및 비트)의 가로 위치는 이 지점 바로 위로 정렬된다. 구현은 두 값이 하나의 공유 상수
  (`SCREW_OFFSET_X`, `static/app.js`)를 참조해 다시 어긋날 수 없는 구조로 되어 있다.
- **투입구 등장 애니메이션 변경**: 새 반제품은 SVG `clip-path`로 투입구 개구부 뒤에 가려진 채
  시작해 이송 단계와 동일한 속도(표준 슬롯 피치 150px, `static/app.js`의 `FEEDER_X = SLOT_X[0] - 150`)로
  이동하며 개구부 경계(`#feederClip`, x=100)를 통과한 부분만 서서히 드러난다. 이 경계는 라인 위
  모든 정지 슬롯의 반제품보다 왼쪽에 있어 정지 상태의 반제품에는 영향이 없다. 목적은 등장 애니메이션과
  기존 반제품의 퇴장 애니메이션이 물리적으로 겹쳐 보이는 순간을 제거하는 것.

## F3. 측정 및 판정 (백엔드)

- 온도·헤드 속도만 측정값 분포에 영향을 준다는 서술은 그대로이나, **레일 속도 관련 서술(F2/F4/공식
  설명에 있던 "레일 속도는 측정값에 영향 없음")은 레일 속도 자체가 사라지면서 조항 전체가 삭제됨**.
- `config.tact_times(head_speed)`로 시그니처 변경(기존 `(head_speed, conveyor_speed)`). 이송 시간은
  `config.INDEX_SEC` 고정값을 그대로 사용한다.

## F4. 엑셀 기록 (openpyxl)

**시트 1 "측정기록" 구조가 완전히 바뀌었다.** 기존(측정 이벤트당 1행, 반제품당 2행) →
**반제품(SN)당 1행**:

| 컬럼 | 내용 |
|---|---|
| 순번 | SN이 처음 측정된 순서 |
| 반제품SN | |
| 토크1(ST1) | ST1 측정값 (NG면 셀 배경 FFC7CE) |
| 판정1 | ST1 판정(OK/NG) |
| ST1시각 | ST1 측정 기록시각 |
| 토크2(ST2) | ST2 측정값 (NG면 셀 배경 FFC7CE) |
| 판정2 | ST2 판정(OK/NG) |
| ST2시각 | ST2 측정 기록시각 |
| 최종판정 | 판정1·판정2 중 하나라도 NG면 NG (두 공정이 모두 기록된 뒤에만 채워짐) |

- LSL/USL은 행마다 반복하지 않고, **1행에 공정별 규격을 1회만 표기**(`ST1(체결토크1): 11.0~13.0 N·m | ST2(체결토크2): 7.2~8.8 N·m`), 2행이 실제 컬럼 헤더, 3행부터 데이터.
- ST1 측정이 먼저 그 SN의 행을 새로 만들고, ST2 측정이 같은 행을 찾아 채운다 (구현은 순서에 무관하게 동작).
- 시트 2 "NG추적", 시트 3 "Lot요약"의 행/컬럼 구조는 변경 없음. 단 Lot요약에서 "레일 속도" 행은 삭제.
- **저장 위치 변경**: `output/LOT-xxx.xlsx` → `config.EXCEL_DIR`
  (`C:\Users\shins\OneDrive\Desktop\Line Simulator\엑셀 데이터`, OneDrive 동기화 경로). 저장 실패 시
  재시도하는 기존 로직(`excel_writer.py`)은 경로만 바뀌었을 뿐 동일하게 적용된다.

## F6. Lot 완료 및 보고서 자동 생성 / F7. Lot 비교 기능

- **보고서 저장 위치 변경**: `output/LOT-xxx_report.html`, `output/COMPARE-*.html` →
  `config.REPORT_DIR` (`C:\Users\shins\OneDrive\Desktop\Line Simulator\보고서`).
  `lots_index.json`은 사용자용 산출물이 아니므로 기존 `./output` 위치를 유지한다(엑셀·보고서와
  분리된 이유).
- **AI 코멘트 구조 변경**: LLM에게 자유 텍스트 대신 `{"analysis": "...", "recommendation": "..."}`
  JSON 응답을 요청한다(`llm_client.py`의 `JSON_RESPONSE_INSTRUCTION`). 파싱 성공 시 보고서는
  해석(`analysis`)을 기존처럼 일반 문단(`.comment-box`)으로, 개선 권고(`recommendation`)를
  별도 강조 블록(`.recommend-box`, 볼드 제목 + 좌측 강조선 + 배경색)으로 렌더링한다. **파싱 실패
  시(모델이 JSON 형식을 안 지키는 경우) 전체 텍스트를 `analysis`에 담고 `recommendation`은 표시하지
  않는 폴백**을 사용하며, 이 경우도 보고서 생성 자체는 계속 진행된다. Lot 보고서·비교 보고서 모두 동일.
- 조건 카드/수치 표에서 "레일 속도" 항목 삭제(F1과 동일 이유). **하위 호환**: `lots_index.json`에
  남아있는 구형 항목(레일 속도 필드 포함)이 있어도 `GET /api/lots`, `GET /api/compare`는 해당 필드를
  단순히 읽지 않으므로 에러 없이 동작한다.

## 9. 최종 인수 기준

- **2번**: "엑셀 파일에 측정기록 40행(20개 × 2공정)이 존재하고" → **"엑셀 파일에 측정기록 20행(2박스
  = 20개, 반제품당 1행)이 존재하고, 모든 행에 토크1·토크2가 채워져 있으며"**. NG 행의 측정값 셀만
  빨간색이라는 조건은 유지(단, "행"이 아니라 "NG인 토크1/토크2 셀"만 빨간색으로 정정).
- **7번(동기화 검증)**: 문구는 유지되나, 두 헤드의 **가로 위치**는 더 이상 동일하지 않다(F2 예외
  조항 참조) — 검증 대상은 어디까지나 **수직 위치**(headY)이며 이는 기존과 동일하게 완전히 일치해야 한다.
- **8번**: 나사 표시 위치 문구에 "체결 헤드의 비트 접촉 지점과 정확히 일치해야 한다"는 조건이 추가됨.
- **신규 인수 기준**: 5박스 연속 생산 애니메이션에서 반제품끼리 시각적으로 겹치는 순간이 한 번도
  없어야 한다(F2 투입 애니메이션 참조).
